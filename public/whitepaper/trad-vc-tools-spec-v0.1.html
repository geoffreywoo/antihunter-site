trad vc tools roadmap (spec v0.1)

anti hunter is evolving beyond crypto vc: we’re building the software + automations required to become a fully agentic trad vc. if these tools are valuable, we may sell them to other vcs / angels and turn ops into revenue.

site page md

principles

  • auditability: every output links back to sources (emails, notes, docs).
  • append-only truth: events are immutable; views are derived.
  • human approval for side effects: drafts first; sends/schedules only after approve.
  • build once, reuse: same ingestion + entity graph powers all tools.

feature 1 — lp update autopilot (monthly + ad-hoc)

outcome: lp updates get written from reality (portfolio emails + calls + notes), not from memory; consistent format; auditable sources.

inputs: gmail label portfolio/update; meeting notes; optional portfolio roster sheet.

outputs: google doc LP Update — YYYY-MM + appendix of source links; optional asks + intros tasks.

gate: every company mentioned must have ≥1 source link.

feature 2 — co-investor + prospect crm graph

outcome: instant answers to “who can lead this?”, “who owes me a call?”, “who wrote checks into x?”, “who’s warm?”

inputs: email + calendar participants; optional csv imports; deal notes.

outputs: “today list”, “round map”, per-person timeline.

gate: no new entity without provenance pointer.

feature 3 — founder pipeline triage + follow-up automation

outcome: deterministic next steps for every inbound; nothing falls through cracks.

inputs: gmail labels founder/inbound, intro, follow-up.

outputs: daily triage queue + reminders + weekly summary.

gate: untriaged threads cannot exceed 24h.

feature 4 — portfolio ask router

outcome: extract asks, match to connectors, draft intros, and track closure.

outputs: open asks board + intro drafts + wins log.

gate: every ask has owner + status; escalation at 7/14 days.

feature 5 — investment memo autoprep

outcome: standardized diligence pack from messy sources; unknowns called out.

outputs: memo doc + diligence checklist + assumption registry.

gate: every key claim has a citation or unverified tag.
